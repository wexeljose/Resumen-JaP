Conceptos Fundamentales de HTML

Una página web es un documento de texto que utiliza marcas, también llamadas etiquetas, para estructurar el contenido y añadir elementos como imágenes o enlaces. Los navegadores web interpretan estas etiquetas para mostrar la página como fue diseñada.

Sintaxis de HTML

La sintaxis de HTML se basa en reglas que definen cómo deben organizarse los elementos. Las etiquetas suelen venir en pares: una de apertura y una de cierre. Por ejemplo, <p> para abrir un párrafo y </p> para cerrarlo.

  • Elementos anidados: Las etiquetas deben anidarse correctamente, sin cruzarse.
  • Elementos vacíos: Algunas etiquetas, como <hr> o <img>, no necesitan cierre.
  • Comentarios: Puedes añadir comentarios que el navegador no mostrará usando la sintaxis <!-- comentario -->.

Estructura Básica de una Página

Una página HTML5 válida tiene una estructura fundamental que incluye la declaración del tipo de documento, la cabecera y el cuerpo. 

<!DOCTYPE html>
<html lang="es">
    <head>
        <meta charset="utf-8">
        <title>Título de la Página</title>
        <link rel="stylesheet" href="estilo.css">
    </head>
    <body>
        <p>Contenido visible de la página.</p>
    </body>
</html>
  • <!DOCTYPE html>: Es una declaración obligatoria que define el documento como HTML5.
  • <html>: Es el elemento raíz que engloba todo el contenido de la página.
  • <head>: Contiene metadatos e información de control, como el título o enlaces a hojas de estilo, que generalmente no se ven en la página.
  • <body>: Incluye todo el contenido visible para el usuario en el navegador.

Atributos, Clases e IDs

Atributos

Los atributos proporcionan información adicional sobre las etiquetas HTML y se especifican en la etiqueta de apertura. Se definen con un nombre y un valor (ej: atributo="valor"). Algunos atributos importantes son:

  • id: Permite identificar un elemento de forma única en toda la página. Ningún otro elemento puede tener el mismo id.
  • class: Se usa para asignar uno o más nombres de clase a un elemento, lo que permite aplicar estilos a múltiples elementos que comparten la misma clase. Un elemento puede tener varias clases, separadas por espacios.
  • style: Permite aplicar estilos CSS directamente a un elemento, aunque se recomienda usar hojas de estilo externas.

Formularios

Un formulario permite al usuario introducir datos y enviarlos a un servidor. Se define con la etiqueta <form>. Sus atributos principales son:

  • action: Indica la URL que procesará los datos del formulario.
  • method: Define cómo se enviarán los datos, siendo "get" o "post" los valores más comunes.

Controles de Formulario

La mayoría de los controles se crean con la etiqueta <input>. Atributos comunes para los controles son:

  • name: Identifica el control para que sus datos puedan ser enviados.
  • value: Define el valor inicial o el valor enviado por el control.
  • required: Hace que rellenar un campo sea obligatorio.
  • placeholder: Muestra un texto de ejemplo en un campo, que desaparece al escribir.
  • disabled: Deshabilita un control para que el usuario no pueda interactuar con él.

Tipos de <input>

  • type="text": Un campo de texto de una línea.
  • type="password": Un campo donde el texto se oculta. No ofrece seguridad en la transmisión de datos, solo oculta la escritura.
  • type="number": Un campo para introducir números, con opciones para definir un valor mínimo (min), máximo (max) y los incrementos (step).
  • type="checkbox": Una casilla de verificación que puede ser marcada o desmarcada.
  • type="radio": Un botón de opción. Varios botones de radio con el mismo atributo "name" forman un grupo, donde solo uno puede ser seleccionado.
  • type="submit": Un botón que envía el formulario.
